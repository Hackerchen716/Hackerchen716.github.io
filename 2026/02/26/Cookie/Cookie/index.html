CTFHub Web — Cookie

题目

打开页面，提示 “hello guest. only admin can get flag.”，说明当前身份是 guest，需要想办法变成 admin。

分析

用 Burp 抓包，在第二个 GET 请求里发现响应头里带了 Set-Cookie: admin=0，说明服务端是靠这个 Cookie 来判断身份的，把值改成 1 就能伪造成 admin。

解法

Burp Suite

在 Repeater 里把请求头中的 Cookie 改为 admin=1，发送请求，响应里直接出 flag。

curl

打开页面的时候浏览器会发两个请求，第一个 GET 服务器返回 302，同时 Set-Cookie: admin=0，浏览器跟着跳到第二个请求才带上 Cookie。用 curl 模拟这个过程：先用 -c 把 Cookie 存下来看看里面有什么，确认字段名是 admin 之后，直接带 admin=1 发请求就拿到 flag 了。

Cookie 伪造是 web 里最经典的入门漏洞之一，服务端把权限判断全靠客户端传来的 Cookie，不做任何验证，改一个字段就能越权。遇到这种题，第一反应就是抓包看 Cookie 里有没有可疑的字段。
