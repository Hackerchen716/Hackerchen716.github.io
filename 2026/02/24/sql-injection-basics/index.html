什么是 SQL 注入？

SQL 注入（SQL Injection）是一种代码注入技术，攻击者通过在应用程序的输入字段中插入恶意的 SQL 代码，从而操控后端数据库执行非预期的查询。

根据 OWASP Top 10，注入漏洞长期位居 Web 应用安全风险榜首。

漏洞成因

当应用程序将用户输入直接拼接到 SQL 查询语句中，而没有进行适当的过滤或参数化处理时，就会产生 SQL 注入漏洞。

常见注入类型

类型 特征 利用方式
联合查询注入 页面有回显 UNION SELECT
报错注入 页面显示数据库错误信息 extractvalue(), updatexml()
布尔盲注 页面只有 True/False 两种状态 逐字符猜测
时间盲注 页面无任何回显 sleep(), benchmark()

防御措施

  1. 使用参数化查询（预处理语句）：这是最有效的防御手段。
  1. 输入验证与过滤：对用户输入进行白名单验证。
  2. 最小权限原则：数据库账户只赋予必要的权限。
  3. WAF：部署 Web 应用防火墙作为额外防护层。
