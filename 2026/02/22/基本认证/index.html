CTFHub Web — 基本认证

题目考点

  • HTTP 基本认证（Basic Auth）
  • Burp Suite Intruder 爆破
  • Base64 编码
  • 脚本爆破

解题思路

题目分析

在 HTTP 中，基本认证（Basic access authentication）是允许 http 用户代理在请求时，提供用户名和密码的一种方式。认证信息以 用户名:密码 的格式经过 Base64 编码后，放在请求头的 Authorization 字段里传给服务器。

打开题目，页面显示 “Here is your flag: click”，点击 click 后弹出浏览器原生的登录框，需要输入用户名和密码。

解题过程

先随便输个账号密码（比如 haha:nihao），用 Burp 抓包，在 Repeater 里可以看到请求头里多了一个 Authorization 字段：

在 Inspector 里点一下，Burp 会自动 Base64 解码，显示 haha:nihao，这就是基本认证的格式：用户名:密码 → Base64 编码。

发送请求后，服务器返回 401 Unauthorized，响应头里有一行：

提示已经很明显了，用户名是 admin，接下来只需要爆破密码。

解法

方法一：Burp Suite Intruder

把请求发到 Intruder，将 Authorization 字段里 Basic 后面的 Base64 部分标记为 payload position，然后配置 Payload：

  • Payload type：Simple list，加载题目提供的密码字典 10_million_password_list_top_100.txt
  • Payload processing：先 Add Prefix admin:，再 Base64-encode
  • Payload encoding：取消勾选 URL-encode，不然 Base64 里的 = 会被转成 %3d 导致认证失败

Start Attack，按 Status 排序，找到状态码为 200 的那条，解码 payload 得到密码是 000000，即 admin:000000。

用 admin:000000 登录，拿到 flag。

方法二：Python 脚本爆破

知道用户名是 admin 之后，也可以直接写脚本跑，逻辑很简单：读字典 → 拼 admin:密码 → Base64 编码 → 塞进请求头 → 判断响应码。

方法三：curl 一行命令

curl 支持 -u 参数直接传用户名密码，会自动处理 Basic Auth 编码：

知道密码之后用这个最省事。

基本认证的问题在于密码只是 Base64 编码，不是加密，任何人抓到包都能直接解出来。遇到弱密码再加上没有频率限制，爆破就是分分钟的事。
