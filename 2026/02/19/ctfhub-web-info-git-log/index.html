CTFHub Web — 信息泄露之 Git 泄露（Log）

发表于2026-02-20|更新于2026-02-20|CTF WriteupWeb安全
|浏览量:

当前大量开发人员使用git进行版本控制，对站点自动部署。如果配置不当,可能会将.git文件夹直接部署到线上环境。这就引起了git泄露漏洞。请尝试使用BugScanTeam的GitHack完成本题

Background

.git 目录暴露在Web服务器上，攻击者可以还原源代码，通过 git log 查看历史提交记录找到敏感信息。

Way1：GitHack

Way2：GitDumper + git-extractor

Way3：手动 curl 下载关键文件

Way4：dirsearch + wget 镜像

Way5：Python脚本自动化

核心思路总结

Flag通常藏在哪？

位置 命令
被删除的文件 git show <旧commit>
提交说明里 git log --pretty=full
stash暂存 git stash pop
其他分支 git branch -a → git checkout other-branch

💡 Log题关键：flag通常在某次历史提交中被”删除”，用 git log 找到可疑commit后用 git show <hash> 或 git checkout <hash> 即可还原！
